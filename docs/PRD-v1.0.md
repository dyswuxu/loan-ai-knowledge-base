# 贷款AI知识库系统 - 产品需求文档（PRD）

**版本**：v1.0
**日期**：2026-07-14
**状态**：草稿

---

## 1. 产品概述

### 1.1 产品定位

面向助贷公司业务人员的AI驱动贷款知识库与客户分析工具。

帮助业务人员快速了解银行产品、评估客户资质、匹配最优贷款方案，降低信息门槛，提升业务效率。

### 1.2 核心价值

| 核心价值 | 说明 |
|---------|------|
| **产品匹配** | 快速找到适合客户的银行+产品组合 |
| **风控辅助** | AI给出分析维度和参考结论，减少判断失误 |
| **知识沉淀** | 自动爬取整理银行产品，不用人工维护 |
| **潜客发现** | 自动爬取有贷款需求的潜在客户线索 |

### 1.3 使用方式

内部工具，无需账号，所有业务人员均可使用。

---

## 2. 目标用户

**主要用户**：助贷公司业务人员

**用户特征**：
- 熟悉贷款业务，但难以掌握全部银行产品细节
- 需要快速判断客户是否符合贷款条件
- 需要发现和跟进潜在贷款客户

## 4. 详细功能说明

### 4.1 客户建档

**功能描述**：录入企业信息 + 上传证件材料，AI自动解析提取关键数据

**录入字段**：
- 企业名称、统一社会信用代码
- 所属行业（制造业/商贸/科技等）
- 从业年限、员工规模
- 年营业额、纳税额
- 有无抵押物（房产/设备等）
- 其他补充信息

**文档上传与解析**：

| 文档类型 | 解析内容 |
|---------|---------|
| 营业执照 | 企业名称、注册资本、经营范围、成立日期 |
| 财务报表 | 营收、利润、资产、负债 |
| 银行流水 | 月均流水、收入稳定性 |
| 房产证 | 房产估值、抵押情况 |

**输出**：结构化的企业画像卡片

---

### 4.2 产品匹配

**功能描述**：根据企业画像，匹配适合的银行贷款产品

**匹配逻辑**：
1. 准入条件过滤（行业、规模、年限等硬指标）
2. 额度预估（根据企业资质推算可贷额度）
3. 综合评分排序（成功率 × 额度 × 利率综合评估）

**展示内容**：
- 推荐产品列表（TOP 3-5）
- 每个产品的匹配度评分
- 关键准入条件 vs 企业实际情况对比
- 预计额度范围、利率范围

---

### 4.3 风控评估

**功能描述**：AI评估企业经营情况和还款能力

**分析维度**：

| 维度 | 指标 |
|------|------|
| 财务健康度 | 营收稳定性、资产负债比、利润率 |
| 经营稳定性 | 从业年限、行业地位、现金流水 |
| 风险预警 | 负面信息（诉讼、失信等）、经营异常 |
| 综合评级 | A/B/C/D 四档 + 参考结论 |

**输出**：风控分析报告卡，包含各项评分、风险点提示、建议措施

---

### 4.4 知识库

**功能描述**：管理和查询银行贷款产品信息

**数据来源**：
- 初始数据：7大银行（工农中建招浦杭）280+产品
- 持续更新：自动爬取银行官网产品信息

**知识库内容**：
- 产品大纲（名称、类型、额度、利率、期限）
- 准入条件（行业、规模、资质要求）
- 担保方式（信用/抵押/质押）
- 特殊条件/优惠政策

**查询方式**：
- 按银行筛选
- 按产品类型筛选（流贷/固贷/信用贷等）
- 按准入条件搜索
- 关键词全文检索

---

### 4.5 潜客发现

**功能描述**：自动爬取有贷款需求的潜在客户线索

**爬取来源**：

| 来源类型 | 示例 |
|---------|------|
| 品牌经销商名单 | 某品牌官网公示的全国经销商列表 |
| 企查查/天眼查 | 按行业、规模筛选有贷款需求的在营企业 |
| 其他公开信息 | 招标公告、中标结果、项目公示等 |

**潜客信息**：
- 企业名称、行业、地区
- 线索来源
- 关联产品推荐
- 跟进状态（待联系/跟进中/已成交/无效）

---

### 4.6 对话窗口

**功能描述**：自然语言交互入口，支持多种查询和操作

**支持场景**：

| 类型 | 示例 |
|------|------|
| 查产品 | "小微企业、有房产抵押、最高能贷多少" |
| 查客户 | "查看今天新建档的客户列表" |
| 问分析 | "帮我分析一下这家企业的风控情况" |
| 做推荐 | "根据这个客户情况，推荐一个产品" |
| 生成报告 | "生成这个客户的贷款方案报告" |

**技术要求**：对话历史保存，支持上下文追问

---

### 4.7 银行产品爬虫（自动采集）

**功能描述**：定时自动爬取银行官网贷款产品信息

**爬取策略**：
- 配置要爬取的银行列表和目标URL
- 解析产品页面的关键字段
- 自动去重和增量更新
- 异常告警（银行产品下架/链接失效等）

**维护方式**：
- 初始配置银行列表和爬取规则
- 定时任务自动执行（如每天凌晨）
- 人工复核 + 确认发布

**数据质量控制**：
- 字段完整性校验
- 与历史数据对比，异常变动告警
- 置信度标记（高置信度自动入库 / 低置信度需人工确认）

---

### 4.8 潜客爬虫（自动采集）

**功能描述**：定时爬取潜在客户线索

**爬取来源**：
- 品牌官网经销商页面（配置品牌名称 + 经销商列表URL）
- 天眼查/企查查搜索结果页
- 其他公开信息源

**爬取策略**：
- 按行业/地区/规模等维度批量爬取
- 自动识别和去重
- 关联推荐可匹配的产品

## 6. 技术架构（概要）

### 6.1 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | Next.js 14 + React + Tailwind CSS |
| 后端 | Next.js API Routes + Node.js |
| 数据库 | PostgreSQL（建议Supabase） |
| 文件存储 | 阿里云OSS / 腾讯云COS |
| AI能力 | MiniMax API（对话+解析+分析） |
| 爬虫 | 自建爬虫 + Puppeteer（JS渲染页面） |
| 定时任务 | Vercel Cron / 或者独立服务器 |

### 6.2 核心API

| 接口 | 说明 |
|------|------|
| `POST /api/customer` | 新建客户档案 |
| `GET /api/customer/:id` | 获取客户详情 |
| `POST /api/customer/:id/documents` | 上传客户证件 |
| `POST /api/analyze` | 综合分析（匹配+风控） |
| `GET /api/products` | 查询产品列表 |
| `POST /api/products/scrape` | 触发产品爬取 |
| `GET /api/potential-customers` | 查询潜客线索 |
| `POST /api/potential-customers/scrape` | 触发潜客爬取 |
| `POST /api/chat` | 对话接口 |

## 7. MVP范围（第一版先做哪些）

### MVP（第一版）

| 功能 | 优先级 | 说明 |
|------|--------|------|

## 9. 风险与依赖

| 风险 | 影响 | 应对 |
|------|------|------|
| 银行产品爬虫被封 | 数据更新中断 | 降级为手动维护，控制爬取频率 |
| 文档解析准确率不足 | 信息提取错误 | 解析结果人工复核入口 |
| 企查查/天眼查反爬 | 潜客数据获取受限 | 考虑付费API或切换数据源 |
